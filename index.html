# Design

How it works

Six principles that make PowerClaw different from asking Claude to write PowerShell directly.

🗂️

Tool registry, not command generation

Claude picks from a known, approved list of tools. It never writes raw PowerShell. The blast radius of any action is defined at design time, not at query time.

🔒

Discovery is not trust

Tools are auto-discovered from disk but only tools explicitly listed in tools-manifest.json become executable. Everything else is ignored.

👁️

Write ops require confirmation

Any tool marked Write risk pauses and shows you exactly what will run before executing. You type Y or N.

🪟

Windows-first

COM, WMI, Task Scheduler, Windows Search index, native Office object models. Not a Linux tool ported to Windows.

🔍

Every run is inspectable

Plans, tool calls, outputs, and errors are logged. Use -Verbose to see the full request/response JSON. Use -Plan to preview without running.

🧩

Drop-in tools

Add a .ps1 file to tools/ with .CLAW_* metadata and add it to the manifest. No core changes needed.

# Setup

Get started

Requires PowerShell 7+ and an Anthropic API key.

1

Clone the repo

# Clone and enter the directory
git clone https://github.com/spinchange/PowerClaw.git
cd PowerClaw
2

Set your API key

# Set for this session
$env:CLAUDE_API_KEY = 'sk-ant-...'

# Or add to your PowerShell profile to persist
3

Import and run

Import-Module .\PowerClaw.psd1
Invoke-PowerClaw -Prompt "What's eating my CPU?"
4

Optional: Playwright (for Fetch-WebPage)

# One-time browser install
$dir = "$env:USERPROFILE\.powerclaw-playwright\PwHost"
mkdir $dir -Force; cd $dir
dotnet new console -n PwHost --framework net10.0 --force
cd PwHost
dotnet add package Microsoft.Playwright
dotnet build
pwsh bin/Debug/net10.0/playwright.ps1 install chromium
# Examples

What to ask

Plain English. Claude figures out which tool — and what arguments — to use.

System Health

  • "Give me a full system health check"
  • "Is my machine under heavy load?"
  • "What's eating all my CPU?"
  • "Which services have stopped unexpectedly?"
  • "What errors happened in the last 24 hours?"
  • "When did my machine last reboot?"

Files & Storage

  • "Find the 10 biggest files on my system"
  • "How much video content do I have?"
  • "What did I download last week?"
  • "Find all zip files over 100MB"
  • "Which drive is closest to full?"
  • "Delete the duplicate installer in Downloads"

Network

  • "What's my external IP?"
  • "What's connected to my network right now?"
  • "Is anything making unexpected connections?"
  • "What DNS servers am I using?"

Web & Files

  • "Summarize the top stories on Hacker News"
  • "Fetch this article and summarize it: [URL]"
  • "Read my config.json and explain my settings"
  • "What does this log file say went wrong?"

Multi-tool Chains

  • "Is my machine healthy? Full diagnostic."
  • "Find big old files in Downloads and clean them up"
  • "Check disk errors in event log, then show storage status"
  • "What process is spiking CPU and is it in the event log?"

Flags

  • -Plan  — preview without running
  • -DryRun — skip write tools
  • -Verbose — see raw API JSON
  • -UseStub — test without API key
